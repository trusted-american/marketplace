# taia-marketplace

Claude Code plugin marketplace maintained by Trusted American Insurance Agency.

## Build & Test
- Install: `npm ci --prefix tools/marketplace-mcp`
- Test MCP: `npm test --prefix tools/marketplace-mcp` (vitest, 57 tests)
- Validate plugins: use `validate_all` MCP tool or CI script

## Tech Stack
- Node.js 22 (MCP server: Hono + Zod + @modelcontextprotocol/sdk)
- pnpm (MCP server lockfile), npm ci for CI
- Vitest for testing
- GitHub Actions CI

## Repository Layout
```
plugins/          — First-party plugins (a3-plugin, playwright)
community/        — Git submodule → third-party plugins
.claude-plugin/   — Auto-generated marketplace.json (DO NOT EDIT)
tools/lib/        — Shared registry generation (registry.js)
tools/marketplace-mcp/  — MCP server (6 tools: create, validate, list, add, conventions)
.github/workflows/      — CI: validate + test + registry update
```

## MCP Tools Available
Use these instead of manual file creation:
- `create_plugin` — scaffold new plugin with correct structure
- `validate_plugin` / `validate_all` — check plugin structure
- `list_plugins` — list all plugins with components
- `add_component` — add skill/agent/command/hook/template to a plugin
- `get_conventions` — full marketplace conventions reference

## Plugin Structure Convention
```
plugin-name/
├── .claude-plugin/plugin.json   ← manifest (only file in .claude-plugin/)
├── skills/skill-name/SKILL.md   ← skills are directories
├── agents/name.md               ← agent definitions
├── commands/name.md             ← slash commands
├── hooks/hooks.json             ← hook configs
├── templates/name.md            ← output templates
├── README.md + LICENSE          ← required
```

## Key Rules
- plugin.json `name` must be kebab-case (`^[a-z0-9-]+$`)
- Components go at plugin root, NEVER inside .claude-plugin/
- marketplace.json is auto-generated by CI — do not edit manually
- Community plugins require `repository` URL in plugin.json
- Skills = directories with SKILL.md; agents/commands/templates = .md files; hooks = .json

## CI Pipeline
1. PR: validate plugin structure + run MCP tests + verify registry matches
2. Merge to main: regenerate marketplace.json, commit with `[skip ci]`

## Don't Touch
- `.claude-plugin/marketplace.json` (auto-generated)
- `tools/marketplace-mcp/node_modules/`
- `community/` submodule content (managed upstream)
