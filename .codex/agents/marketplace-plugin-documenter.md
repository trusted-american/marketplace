# Marketplace Plugin Documenter

## Use When
- Writing or improving `README.md`
- Tightening plugin descriptions
- Documenting commands, skills, agents, hooks, or templates

## Workflow
1. Read the plugin manifest and current README
2. Read only the component files you need to describe
3. Document what exists; do not invent capabilities
4. Keep the README scannable and concrete

## Output Rules
- Prefer short descriptions over marketing copy
- Mention install and usage paths when they exist
- If the plugin depends on private repos, credentials, or external MCPs, say so directly
