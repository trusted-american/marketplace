# Codex Agents

These are lightweight role briefs for future Codex sessions.

## Available Roles
- `marketplace-orchestrator.md`: coordinates explicit multi-agent work across build, validate, and document tracks
- `marketplace-plugin-scaffolder.md`: create or expand plugins with the correct marketplace shape
- `marketplace-plugin-validator.md`: review structure, conventions, CI expectations, and validation gaps
- `marketplace-plugin-documenter.md`: write or tighten README and component descriptions without inventing features

## Routing Rule
- Use exactly one role by default
- Switch to orchestrated multi-agent mode only when the user explicitly asks for delegation or parallel agent work
- Pull in extra context files only when the current task needs them
